# 🗄️ Storacha IPFS Logger

This component is responsible for the "Auditability" pillar of FlowTalos. Every time the AI agent makes a decision, a reasoning log is generated. This module ensures that log is immutable and verifiable.

## 🏗️ Architecture Role
1. **Hashing:** Takes the JSON reasoning log generated by the Python agent.
2. **CID Generation:** Computes a unique Content-Addressed ID (CIDv1 using SHA-256) via the `multiformats` library.
3. **Decentralized Storage:** (Optional) Pins the raw JSON to the Storacha / Web3.Storage network for permanent public access.

This generated CID is then injected into the on-chain transaction logs, creating an unbreakable cryptographic link between an on-chain trade and the off-chain AI reasoning that caused it.

## 🚀 Setup

```bash
cd storacha-logger
npm install
```

If you wish to actually upload files to the IPFS network (rather than just computing the CID locally), you must configure `STORACHA_EMAIL_ADDRESS` and `STORACHA_SPACE_DID` in your root `.env` file.
