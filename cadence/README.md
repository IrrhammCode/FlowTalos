# ⛓️ FlowTalos Smart Contracts (Cadence)

This directory contains the foundational Cadence smart contracts for the FlowTalos Protocol, enabling trustless, scheduled AI execution on the Flow blockchain.

## 🏗️ Architecture Role
The contracts here manage the intersection of AI intent, user funds, and on-chain execution. We heavily utilize the **Flow Transaction Scheduler** pattern.

### Key Contracts:
- `FlowTalosVault.cdc`: Manages the AI's Cadence Owned Account (COA). It exposes a secure `executeEVMCalls` function that takes batched EVM transactions generated by the AI agent and executes them trustlessly on Flow EVM.
- `FlowTalosStrategyHandler.cdc`: Implements the `TransactionHandler` interface. This is the contract that the Flow Network calls when an AI-scheduled transaction matures. It delegates execution to the Vault.

### Key Transactions:
- `InitVaultAndHandler.cdc`: Idempotent setup script that provisions the VaultManager and Scheduler Handler on the user's account.
- `ScheduleAIStrategy.cdc`: The transaction the AI Agent signs (via Lit Protocol) to schedule future execution. It includes dynamic delay computation and fee estimation.

## 🛠️ Deployment (Testnet)

We deploy to the Flow Testnet.

```bash
# Generate keys if you haven't already
flow keys generate

# Deploy the project
flow project deploy --network=testnet
```

For full environment setup, please see the `.env.example` in the root directory.
